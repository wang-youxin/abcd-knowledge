## 规则（优先级从高到低）

### 规则 0：对话自动归档（每次对话强制执行）

**每次收到用户消息后，必须执行以下两步，不需要用户说任何指令：**

1. 追加到 `raw/完整对话/YYYY-MM-DD.md`：
   ```
   **HH:MM 用户** — {用户消息原文}
   **HH:MM AI** — {AI 回复原文，逐字，不缩写不概括}
   ```
2. 追加到 `raw/用户消息/YYYY-MM-DD.md`：
   ```
   **HH:MM** — {用户消息原文}
   ```

当天第一条消息先创建文件，后续追加。

### 规则 1：raw/ 只放不读

**绝对禁止读取 raw/ 内的任何内容。** 唯一的例外：用户明确说"扫一下 raw，更新 wiki"。

raw/ 是仓库，不是工作台。AI 只能往里写，不能从里读。

### 规则 2：每次修改/新增 wiki、skills、daily 后更新相关索引

### 规则 3：所有交叉引用使用 `[[wikilink]]` 双链语法（Obsidian 兼容）

### 规则 4：同一方法论在 wiki 中出现 3 次以上 → 自动封装为 C 层技能

### 规则 5：绝不直接删除 Vault 内任何文件

删除、替换、重构一律移入 `废弃内容/` 文件夹。

### 规则 6：每周复盘检查飞轮循环

检查标准：
- A 层有新料？
- B 层有更新？
- C 层有新封装？
- D 层有回流？

## 四层说明

### A 层 — raw/（素材层）

- **只放不读。** AI 绝对禁止主动读取。
- 用户扔进来的文章、网页、对话记录，原文存入，永不可改。
- 不用分类、不用提炼、不用改格式。
- 对话消息自动归档进 `raw/完整对话/` 和 `raw/用户消息/`。

### B 层 — wiki/（提炼层）

- AI 扫描 raw/ 新素材（仅在用户说"扫一下 raw 更新 wiki"时），提炼关键概念。
- 写成 `[[双方括号]]` 互链的笔记。
- AI 自动发现概念间关系，补链接、更新交叉引用。
- 每个页面至少有一个 wikilink——不做孤岛页面。

### C 层 — skills/（技能层）

- 从 wiki 中反复出现 3 次以上的经验中提炼。
- 格式：什么场景 + 怎么做 + 为什么有效 + 什么时候不适用。
- 每一条都是经过多轮验证的、用户自己用出来的方法论。
- 技能用过发现问题要回溯修改，不只封装不更新。

### D 层 — daily/（产出层）

- 每天工作日志 `YYYY-MM-DD.md`。
- 包含：做了什么、做了什么决定、踩了什么坑、有什么发现。
- 日志里的坑和教训自动回流给 B 层。
- D 层的新认知必须回到 B 层——不产出后不回流。

## 用户指令

| 指令 | 行为 |
|------|------|
| **"记下来"** | 把刚才聊的结论、踩的坑、做的决定，存成结构化笔记，放进对应目录（wiki/ daily/ skills/），加 `[[链接]]`，更新相关索引 |
| **"今天结束"** | 扫描全天对话 → 写 `daily/YYYY-MM-DD.md`。坑和教训回流 wiki |
| **"扫一下 raw，更新 wiki"** | 扫描 raw/ 新素材 → 提炼概念 → 写/更新 wiki 互链笔记。汇报新增/更新/新关联 |
| **"/lint"** | 检查死链、孤儿页面、过期信息 |

## 禁止行为（Anti-patterns）

- 不要在 raw/ 里直接写笔记或读取 raw/
- 不要让 wiki/skills 页面变成孤岛——每个页面至少有一个 wikilink
- 不要产出后不回流——D 层新认知必须回到 B 层
- 不要只封装不更新——技能用过发现问题要回溯修改
