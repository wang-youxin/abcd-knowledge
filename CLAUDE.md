# CLAUDE.md — ABCD 四层知识复利系统

## 项目概述

这是我的 Obsidian 知识库，用 AI 帮我像盖楼一样一层一层堆知识。四层结构：A 层扔素材 → B 层 AI 提炼成知识网络 → C 层把反复验证的经验变成方法论 → D 层是每天产出和踩坑，坑和教训回流给 B 层。

## 工作目录

`C:/Users/lenovo/Documents/Obsidian Vault/`

## 四层目录结构

```
Obsidian Vault/
├── raw/        A 层 素材层 — 只扔不整理。所有对话消息自动存入 raw/对话记录/
│   └── 对话记录/   ← 每次对话自动归档，格式 YYYY-MM-DD.md
├── wiki/       B 层 提炼层 — AI 从 raw 提炼的互链笔记，用 [[双方括号]] 链接
├── skills/     C 层 技能层 — 从 wiki 反复出现的经验中提炼的可执行方法论
├── daily/      D 层 产出层 — 每天的工作日志、踩坑记录、决定
└── CLAUDE.md   ← 本文件
```

## 每条目录的规则

### raw/ — A 层素材层
- **只放不读。** AI 只能往 raw/ 写入，绝不能读取 raw/ 内容，除非用户明确说"扫一下 raw 更新 wiki"。
- 好文章、网页、对话记录直接扔进来。不用分类、不用提炼、不用改格式。
- 别替未来的自己做决定——今天觉得没用的东西，3 个月后可能是金矿。
- 所有对话消息自动归档进 raw/，但 AI 回复时不准参考 raw/ 里的任何内容。

### wiki/ — B 层提炼层
- AI 扫描 raw/ 的新素材，提炼关键概念写成 [[双方括号]] 链接的笔记。
- AI 自动发现概念之间的关系，补链接、更新交叉引用。
- 新知识不是孤零零落进去，是嵌进一张已经在生长的网里。

### skills/ — C 层技能层
- 从 wiki 中反复出现 3 次以上的经验中提炼。
- 每一条都是经过多轮验证的、自己用出来的方法论。
- 格式：一条方法 = 什么场景 + 怎么做 + 为什么有效 + 什么时候不适用。

### daily/ — D 层产出层
- 每天的工作日志。文件名格式：`YYYY-MM-DD.md`
- 包含：今天做了什么、做了什么决定、踩了什么坑、有什么发现。
- 日志里的坑和教训，AI 下一次整理 B 层时会自动回流进 wiki。

## 关键约定

- **raw/ 只放不读** — AI 绝对不能主动读取 raw/ 内容，除非用户明确说"扫一下 raw 更新 wiki"
- Obsidian 兼容：正文引用用 `[[文件名]]`
- 笔记之间用双链，不孤立
- 别手动整理——让 AI 干。你只负责扔素材和说"记下来"
- 不要删笔记，不想要了扔进 `废弃/` 文件夹

## 四条自动规则

### 0. 对话归档（自动，每次对话都执行，存入 A 层）

**每条消息都要自动归档，两条线并行：**

- **完整对话** → `raw/完整对话/YYYY-MM-DD.md` — 用户和 AI 的每一句话，原文逐字记录，不缩写不概括
- **用户消息** → `raw/用户消息/YYYY-MM-DD.md` — 从完整对话中复制，只保留用户的部分

格式：
```
**HH:MM 用户** — {用户消息原文}
**HH:MM AI** — {AI 回复原文}
```

**必须当场存，不准事后补。** 事后记不住原文，必然变成摘要。
不需要我说任何指令，自动执行。对话本身就是 A 层原始素材。

### 1. "记下来"
把我刚才聊的结论、踩的坑、做的决定，自动存成结构化笔记，放进对应目录（wiki/ 或 daily/ 或 skills/），加上 `[[链接]]`，更新相关索引。

### 2. "今天结束"（或 "今天结束了"）
扫描今天一整天的对话，提取：
- 今天做了什么
- 做了什么决定
- 踩了什么坑
- 有什么发现
写成一篇工作日志，保存为 `daily/YYYY-MM-DD.md`。
日志里如果出现"X 做法行不通，因为 Y"这类教训，自动去 wiki/ 找到相关页面加上标注，防止下次再踩。

### 3. "扫一下 raw 更新 wiki"
扫描 raw/ 中新的素材（过去一周内新增的），提炼关键概念，写成/更新 wiki/ 中的互链笔记。自动发现概念之间的关联并补上 `[[链接]]`。每次更新后告诉我：新增了哪些页面、更新了哪些页面、新发现了哪些关联。

### 4. "/lint"
检查死链、孤儿页面、过期信息。

## 六条 Rules

1. **绝不编辑 raw/** — 原始素材永远只读
2. **每次修改/新增文件后更新 wiki/index.md**
3. **所有交叉引用使用 [[wikilinks]] 语法**
4. **同方法论使用 3 次 → 自动封装 C 层技能**
5. **每周复盘检查循环是否在转**（A 有新料？B 有更新？C 有封装？D 回流了？）
6. **绝不直接删除 Vault 内任何文件** — 删/替/重构一律移入 `废弃内容/`

---

## Anti-patterns

- 不要在 raw/ 里直接写笔记
- 不要让 wiki/skills 页面变成孤岛 — 每个页面至少有一个 wikilink
- 不要产出后不回流 — D 层的新认知必须回到 B 层
- 不要只封装不更新 — 技能用过发现问题要回溯修改
