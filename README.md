# ABCD — 四层知识复利系统

**让你的 AI 从每次从零开始，变成每天都在盖楼。**

---

## 问题

你用 AI 聊了大半年，每次打开对话框，上次聊透的结论、磨出来的思路全没了。好像每次都是重新开始。半年下来，知识根本没攒下多少。

这不是你的问题，是用 AI 的方式不对。

OpenAI 联合创始人 Karpathy 2025 年说过一句话：

> **不要把 AI 当成每次重建上下文的检索工具。要把 AI 当成持续维护知识库的编译器。**

检索 = 每次从平地挖坑。编译 = 在已有的基础上盖第二层、第三层。

ABCD 就是这个"编译器"。

---

## 是什么

ABCD 是一套让 AI 自动帮你积累知识的规则系统。四层结构，像盖楼一样一层一层往上搭：

| 层 | 目录 | 做什么 | 你做什么 |
|----|------|--------|----------|
| **A** | `raw/` | 所有对话自动归档，素材只扔不整理 | 零操作，自动的 |
| **B** | `wiki/` | AI 扫描 raw，提炼成 [[双链]] 互链知识网络 | 说一句"扫一下 raw，更新 wiki" |
| **C** | `skills/` | 反复出现 3 次的方法论自动封装成技能 | 零操作，自动的 |
| **D** | `daily/` | 每天工作日志，踩的坑回流到知识网络 | 说一句"今天结束" |

**飞轮**：A 扔素材 → B 提炼成网 → C 封装方法 → D 产出+踩坑 → 坑回流 B → 循环。

## 安装

### 方式 1：作为 Skill 安装

```bash
npx skills add {owner}/abcd-knowledge -g -y
```

然后在 Claude Code / Hermes / Codex 中说：

```
搭建 ABCD 知识库，放在 ~/Documents/MyVault/
```

AI 会自动创建四层目录结构并写入规则文件。

### 方式 2：手动安装

```bash
git clone https://github.com/{owner}/abcd-knowledge.git
cp -r abcd-knowledge/scaffold/* ~/Documents/MyVault/
```

### 方式 3：给其他 AI 用

把 `AGENTS.md` 复制到你的知识库根目录。任何能读文件的 AI 都能遵守这套规则。

---

## 日常使用

每天只需要说三句话：

| 你说 | AI 做 |
|------|-------|
| （不用说，自动的） | 每次对话自动归档到 `raw/` |
| **"记下来"** | 把刚才聊的结论存成笔记 |
| **"今天结束"** | 扫全天对话 → 写日志，踩坑回流 |
| **"扫一下 raw，更新 wiki"** | AI 提炼新素材为知识网络（隔一两周一次） |

---

## 三周效果

- **第一周**：每天对话自动存下来了，但感觉不到明显变化
- **一个月**：搜个关键词，弹出一条你完全不记得写过的笔记，它已经在那儿等你了
- **三个月**：AI 不再每次从零认识你。它真的懂你的知识体系

---

## 不是什么

- 不是笔记软件替代品——是让 AI 帮你用笔记软件
- 不是另一个需要维护的系统——你只负责说三句话
- 不是固定流程——你随时可以直接跟 AI 聊天，归档自动进行

## 文件结构

```
abcd-knowledge/
├── SKILL.md           ← Skill 定义（AI 读）
├── AGENTS.md          ← 通用规则模板（任何 AI 读）
├── README.md          ← 本文件（人读）
└── scaffold/          ← 初始目录结构模板
    ├── raw/
    │   ├── 完整对话/
    │   └── 用户消息/
    ├── wiki/
    ├── skills/
    ├── daily/
    └── 废弃内容/
```
